# AGENTS.md — OpenSpec

## Purpose

Owns OpenSpec project context, active changes, archived changes, and capability specs.

## Ownership

- `project.md`: OpenSpec project-level guidance.
- `changes/`: proposed or in-progress changes with proposals, designs, tasks, and scoped spec deltas.
- `specs/`: accepted capability specs.

## Local Contracts

- Avoid editing this subtree unless the task is explicitly about specs, proposals, OpenSpec tooling, or a change tracked here.
- Keep proposals, tasks, and spec deltas consistent when a change is modified.
- Do not treat archived changes as active work unless explicitly requested.

## Work Guidance

- Use `.opencode/` OpenSpec commands/skills when the task is an OpenSpec workflow and the user requests that path.
- Preserve OpenSpec structure and naming.

## Verification

- Use the relevant OpenSpec verification command or skill for the changed artifact when applicable.

## Child DOX Index

- `changes/`: active and archived change packages.
- `specs/`: accepted specs by capability.
