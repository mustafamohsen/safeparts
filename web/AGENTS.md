# AGENTS.md — Web App

## Purpose

Owns the Vite + React browser UI, web packaging, WASM build integration, and local web tests.

## Ownership

- `src/`: React application, components, hooks, i18n, styles, and generated WASM package boundary.
- `scripts/build-wasm.sh`: builds `crates/safeparts_wasm` for browser consumption.
- `tests/`: Playwright/axe browser tests.
- `help/`: Astro + Starlight documentation site; see its child AGENTS.md.
- `Dockerfile`, `nginx.conf`, `public/`, `cloudflare-worker.ts`, and package config for static deployment.

## Local Contracts

- The web UI must keep split/combine workflows local to the browser; do not introduce a backend for secrets.
- `src/wasm_pkg/` is generated by `bun run build:wasm`; avoid hand-editing generated artifacts except when explicitly required.
- User-facing docs under `help/` require the `humanizer` skill before finalizing prose edits.

## Work Guidance

- Follow `docs/agents/conventions.md` for React/TypeScript style and user-friendly error handling.
- Use the `frontend-design` skill for UI construction or substantial visual redesign.
- For browser-based Web UI checks, use the `browser` skill / `browse` CLI in local mode unless the user explicitly asks for Playwright.

## Verification

- Install: `bun install`
- Build WASM: `bun run build:wasm`
- Build app: `bun run build`
- Cloudflare Worker dry run from repo root: `bunx wrangler deploy --dry-run --config wrangler.jsonc`
- Dev server: `bun run dev`
- Existing test scripts: `bun run test:e2e:smoke`, `bun run test:e2e:full`
