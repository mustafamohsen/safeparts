The web app runs at the site root and performs split and combine in the browser through WASM. It does not need a Safeparts server for cryptographic work.

<LinkButton href="/" icon="external" target="_blank" rel="noopener noreferrer">
	Open the web app
</LinkButton>

## What the web app exposes

- Split and combine for text secrets entered in the browser UI.
- Recovery plans such as 2 of 3 or 3 of 5, with up to 255 total shares.
- Two share formats: <span class="token">Words</span> (<span class="token">mnemo-words</span>) and <span class="token">Letters</span> (<span class="token">base64url</span>).
- Optional passphrase protection.
- Best-effort format detection when you paste shares into the combine form.

For binary files or all encoding formats, use the [CLI](../cli/) or [TUI](../tui/).

## Split

<Steps>
1. Open the <span class="token">Split</span> tab.
2. Enter a secret.
3. Choose how many shares are required and how many total shares to create.
4. Choose <span class="token">Words</span> or <span class="token">Letters</span>.
5. Add a passphrase only if your plan includes a separate passphrase custody path.
6. Click <span class="token">Split</span>.
7. Store each share in a separate place.
</Steps>

## Combine

<Steps>
1. Open the <span class="token">Combine</span> tab.
2. Paste one share per box. The app can add boxes after reading share metadata.
3. Confirm the detected or selected format.
4. Enter the passphrase if the split used one.
5. Click <span class="token">Combine</span>.
6. Copy the recovered secret only into the system that needs it.
</Steps>

## Privacy boundary

The default hosted web app runs locally in your browser. Your secret can still leave your device if you copy it into another service, use a modified build, install a browser extension that reads page content, or let screen capture tools record the workflow.

For sensitive recovery, consider an offline machine and verify that your storage plan keeps shares separated.
