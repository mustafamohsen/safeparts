يعمل تطبيق الويب في جذر الموقع وينفّذ التقسيم والاسترداد داخل المتصفح عبر WASM. لا يحتاج إلى خادم Safeparts لتنفيذ العمل التشفيري.

## ما الذي تعرضه واجهة الويب

- التقسيم والاسترداد للأسرار النصية المدخلة في الواجهة.
- خطط استرداد مثل 2 من 3 أو 3 من 5، مع ما يصل إلى 255 حصة إجمالاً.
- صيغتان للحصص: <span class="token">Words</span> وهي <span class="token" dir="ltr">mnemo-words</span>، و<span class="token">Letters</span> وهي <span class="token" dir="ltr">base64url</span>.
- حماية اختيارية بعبارة مرور.
- كشف best-effort للصيغة عند لصق الحصص في نموذج الاسترداد.

للملفات الثنائية أو كل صيغ الترميز، استخدم [CLI](../cli/) أو [TUI](../tui/).

## التقسيم

<Steps>
1. افتح تبويب <span class="token">تقسيم</span>.
2. أدخل السر.
3. اختر عدد الحصص المطلوبة وعدد الحصص الإجمالي الذي تريد إنشاءه.
4. اختر <span class="token">Words</span> أو <span class="token">Letters</span>.
5. أضف عبارة مرور فقط إذا كانت الخطة تتضمن مسار حيازة منفصل لعبارة المرور.
6. اضغط <span class="token">تقسيم</span>.
7. خزّن كل حصة في مكان منفصل.
</Steps>

## الاسترداد

<Steps>
1. افتح تبويب <span class="token">استعادة</span>.
2. الصق حصة واحدة في كل مربع. يمكن للتطبيق إضافة مربعات بعد قراءة بيانات الحصة.
3. تأكد من الصيغة المكتشفة أو المختارة.
4. أدخل عبارة المرور إذا استخدم التقسيم واحدة.
5. اضغط <span class="token">استعادة</span>.
6. انسخ السر المستعاد فقط إلى النظام الذي يحتاجه.
</Steps>

## حدود الخصوصية

ينفّذ تطبيق الويب المستضاف افتراضياً العمل محلياً داخل المتصفح. مع ذلك يمكن أن يغادر السر جهازك إذا نسخته إلى خدمة أخرى، أو استخدمت build معدلاً، أو ثبّت إضافة متصفح تقرأ محتوى الصفحة، أو سمحت لأدوات تسجيل الشاشة بتسجيل سير العمل.

للاسترداد الحساس، فكّر في جهاز دون اتصال وتحقق من أن خطة التخزين تفصل الحصص.
