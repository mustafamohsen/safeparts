هذه الصفحة للمهندسين والمراجعين الذين يحتاجون إلى فهم التصميم خلف سير المستخدم. للإرشاد التخطيطي، ابدأ بـ [نموذج الأمان](../security/) و[اختيار خطة استرداد](../use-cases/).

## تدفق البيانات

يعمل Safeparts على البايتات.

```text
Split
-----
secret bytes
  |
  | optional passphrase protection
  |   Argon2id(passphrase, salt, params) -> key
  |   ChaCha20-Poly1305(key, nonce) -> ciphertext
  |
  | data_to_split = plaintext or ciphertext
  |
  | BLAKE3(data_to_split) -> 32-byte tag
  | tagged = data_to_split || tag
  |
  | Shamir split over GF(256), byte by byte
  |
  | SharePacket per share
  |
  +-> text encoding for storage or transport
```

```text
Combine
-------
text input -> SharePacket values
  |
  | validate set metadata, share indexes, and crypto params
  |
  | Shamir combine over GF(256)
  |
  | split reconstructed bytes into data_to_split and tag
  | verify BLAKE3(data_to_split) == tag
  |
  | if encrypted:
  |   Argon2id(passphrase, salt, params) -> key
  |   ChaCha20-Poly1305 decrypt -> secret bytes
  |
  +-> recovered secret bytes
```

## مشاركة العتبة

يستخدم Safeparts مشاركة أسرار بأسلوب Shamir فوق <span class="token token--mono" dir="ltr">GF(256)</span>، وتُطبّق بشكل مستقل على كل موضع بايت.

لكل بايت في payload الموسوم:

1. اعتبر البايت الحد الثابت في كثيرة حدود.
2. اختر عشوائياً <var>k</var>-1 معاملات.
3. قيّم كثيرة الحدود عند إحداثيات الحصص <span class="token" dir="ltr">x = 1..n</span>.
4. خزّن بايت النتيجة في payload الحصة.

Payload كل حصة له طول payload الموسوم نفسه. يستخدم الاسترداد أي <var>k</var> إحداثيات فريدة من نفس set ويستوفي كل بايت عند <span class="token" dir="ltr">x = 0</span>.

مجال النواة هو <span class="token" dir="ltr">1 &lt;= k &lt;= n &lt;= 255</span>. يحتوي الحقل على 256 قيمة ويحجز Safeparts <span class="token" dir="ltr">x = 0</span> للاسترداد.

## حزم الحصص

`SharePacket` هو الغلاف ذي الإصدار حول حصة خام واحدة. يحمل بيانات غير سرية مطلوبة لرفض الأخطاء الشائعة.

شكل الحزمة الثنائية الحالية، مبسط:

```text
magic       : "SMN1"
version     : u8
flags       : u8
k, n, x     : u8, u8, u8
set_id      : 16 bytes

if encrypted:
  salt          : 16 bytes
  nonce         : 12 bytes
  argon memory  : u32 KiB
  argon time    : u32
  argon lanes   : u32

payload_len : u32
payload     : bytes
```

يمكن لحزم الإصدار 2 حمل معلمات التشفير. تبقى حزم الإصدار 1 قابلة للتحليل للتوافق حيث يدعمها parser الخاص بالحزم.

بيانات الحزمة ليست سرية. تأتي السرية من مشاركة العتبة ومن التشفير قبل التقسيم عند تفعيله.

## وسم السلامة

يمكن لمشاركة Shamir أن تعيد بناء خرج من مدخلات خاطئة. لذلك يضيف Safeparts وسم BLAKE3 بطول 32 بايت إلى البيانات قبل التقسيم حتى يرفض combine الحصص التالفة أو المختلطة.

يكشف الوسم:

- تلف النص أو بايتات الحزمة؛
- حصصاً من sets تقسيم مختلفة؛
- أخطاء نسخ تمر من طبقة الترميز النصي؛
- نصاً مشفراً معاد البناء لا يطابق payload المشفر الأصلي.

وسم BLAKE3 ليس MAC بمفتاح. هو فحص اتساق داخلي، وليس دليلاً على أن جهة موثوقة أنشأت الحصص.

## حماية عبارة المرور

تحمي عبارة المرور السر بالتشفير مرة واحدة قبل التقسيم. لا تظهر عبارة المرور داخل الحزمة.

الاشتقاق والتشفير الحاليان:

- KDF: Argon2id.
- الملح: 16 بايت عشوائية لكل تقسيم.
- تكلفة الذاكرة الافتراضية: 65,536 KiB.
- تكلفة الزمن الافتراضية: 3.
- التوازي الافتراضي: 1.
- AEAD: ChaCha20-Poly1305.
- nonce: 12 بايت عشوائية لكل تقسيم.

إذا حصل مهاجم على <var>k</var> حصص مشفرة، يستطيع إعادة بناء النص المشفر ومحاولة تخمين عبارة المرور offline. يرفع Argon2id تكلفة هذا الهجوم، لكن عبارة المرور الضعيفة تبقى ضعيفة.

يتحقق Safeparts من وسم BLAKE3 قبل فك التشفير. في التقسيمات المشفرة، يتحقق ذلك من النص المشفر المعاد بناؤه. بعد ذلك يرفض ChaCha20-Poly1305 عبارات المرور الخاطئة أو النصوص المشفرة المعدلة.

## الترميزات النصية

يمكن ترميز الحزمة الثنائية كالتالي:

- <span class="token" dir="ltr">base64url</span>
- <span class="token" dir="ltr">base58check</span>
- <span class="token" dir="ltr">mnemo-words</span>
- <span class="token" dir="ltr">mnemo-bip39</span>

طبقة الترميز تشغيلية. تساعد التخزين والنقل والإدخال اليدوي. لا تغيّر مخطط العتبة الأساسي.

راجع [الترميزات](../encodings/) لاختيار الصيغة ومعرفة دعم الواجهات.

## سلوك الفشل

يفشل الاسترداد عندما:

- يتم توفير أقل من <var>k</var> حصص؛
- تتكرر إحداثيات الحصص؛
- تختلف بيانات الحزمة؛
- تختلف معلمات التشفير بين الحصص المشفرة؛
- لا يطابق وسم السلامة المعاد بناؤه؛
- تتطلب الحصص المشفرة عبارة مرور ولم يتم توفيرها؛
- يفشل فك التشفير.

يجب على التطبيقات عرض نوع الخطأ دون تسجيل نص الحصص أو عبارات المرور أو البايتات المستعادة.

## الحدود

- لا توجد طبقة تخزين. لا يقرر Safeparts أين تعيش الحصص.
- لا توجد مشاركة أسرار قابلة للتحقق VSS. يمكن لحامل خبيث تقديم حصة خاطئة وتعطيل الاسترداد.
- لا يوجد refresh للحصص دون إعادة بناء السر وإنشاء تقسيم جديد.
- لا حماية من نصاب شرعي. أي شخص يملك <var>k</var> حصص صالحة وعبارة المرور عند استخدامها يستطيع الاسترداد.
- لا توجد دلالات seed لمحافظ العملات. صيغ mnemonic هي ترميزات لحصص Safeparts، وليست seeds.
