يعيد التقسيم قيماً ثنائية من نوع `SharePacket`.
معظم التطبيقات يجب أن ترمّز هذه الحزم قبل تخزينها أو إرسالها للمستخدمين أو كتابتها في runbook.

## ترميز الحزم

استخدم `safeparts_core::encoding` لواجهة الترميز العامة.

```rust
use safeparts_core::encoding::{self, Encoding};
use safeparts_core::{split_secret, CoreResult};

fn encode_for_storage() -> CoreResult<Vec<String>> {
    let packets = split_secret(b"example-secret", 2, 3, None)?;

    packets
        .iter()
        .map(|packet| encoding::encode_packet(packet, Encoding::Base64url))
        .collect()
}
```

`Encoding::Auto` صالح للقراءة والتحليل، لا للإخراج.
إذا حاولت الترميز بـ `Auto` فسترجع المكتبة خطأ لأن الإخراج يحتاج صيغة محددة.

## تحليل نص الحصص

لإدخال المشغلين، تقبل `parse_share_packets` نص حصص ملصقاً وتعيد الحزم والترميز المكتشف.

```rust
use safeparts_core::encoding::{self, Encoding};
use safeparts_core::{combine_shares, CoreResult};

fn recover_from_paste(input: &str) -> CoreResult<Vec<u8>> {
    let parsed = encoding::parse_share_packets(input, Encoding::Auto)?;
    combine_shares(&parsed.packets, None)
}
```

إذا كانت واجهتك تعرف الصيغة مسبقاً، مرر الترميز المحدد بدلاً من `Auto`.
هذا يعطي أخطاء أوضح ويتجنب التخمين عندما يكون الإدخال تالفاً.

## اختيار الترميز

`base64url` مضغوط ومناسب للملفات وواجهات API والنسخ واللصق بين الأنظمة.

`base58check` مضغوط أيضاً، لكنه يتجنب المحارف الملتبسة ويضيف checksum في طبقة النص.
يفيد عندما قد يقرأ المشغلون الحصص أو يكتبونها يدوياً.

`mnemo-words` يستخدم كلمات مع CRC16.
اختره لسير العمل الورقي والنسخ اليدوي، خصوصاً إذا كان المستخدم سيقرأ حصة عبر مكالمة.

`mnemo-bip39` ينتج عبارات صالحة وفق BIP-39.
استخدمه فقط عندما تحتاج هذا التوافق. حصة Safeparts ذات الكلمات ليست seed لمحفظة.

## معنى بيانات الحزمة

تحمل كل حزمة بيانات تكفي للاسترداد الآمن:

- `k` و`n`: العتبة وعدد الحصص الكلي
- `x`: إحداثية الحصة
- `set_id`: معرّف عشوائي لمجموعة التقسيم
- `payload`: بايتات الحصة
- `crypto_params`: معاملات التشفير الاختيارية

هذه البيانات ليست سرية.
تساعد مسار الاسترداد على رفض خلط المجموعات، وتكرار الحصص، ومعاملات التشفير غير المتوافقة.

## نمط تخزين شائع

نمط التطبيق المعتاد هو:

1. تقسيم البايتات إلى قيم `SharePacket`
2. ترميز كل حزمة بترميز محدد
3. تخزين كل نص حصة في نطاق فشل منفصل
4. لاحقاً، تحليل <var>k</var> حصص مشفرة على الأقل
5. دمج الحزم والتعامل مع البايتات المستردة كمخرجات حساسة

احتفظ باسم الترميز مع سجل التخزين إذا لم يكن المشغلون سيلصقون كل الحصص عبر الاكتشاف التلقائي.
هذا يجعل الاسترداد لاحقاً أقل هشاشة.
