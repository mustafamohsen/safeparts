يجب أن تثبت الاختبارات حدود التكامل دون استخدام أسرار إنتاجية. سلاسل بايت اصطناعية تكفي لتغطية التقسيم والاسترداد والترميز والتلف وسلوك عبارة المرور.

## اختبار سلوك العتبة

```rust
use safeparts_core::{combine_shares, split_secret, CoreError};

#[test]
fn round_trip_with_any_k_shares() {
    let shares = split_secret(b"synthetic-secret", 2, 3, None).unwrap();

    let recovered = combine_shares(&[shares[0].clone(), shares[2].clone()], None).unwrap();

    assert_eq!(recovered, b"synthetic-secret");
}

#[test]
fn insufficient_shares_fail() {
    let shares = split_secret(b"synthetic-secret", 3, 5, None).unwrap();

    let err = combine_shares(&shares[..2], None).unwrap_err();

    assert!(matches!(err, CoreError::NotEnoughShares { .. }));
}
```

## اختبار حدود الترميز

إذا كان تطبيقك يخزّن حصصاً نصية، اختبر الترميز والتحليل معاً.

```rust
use safeparts_core::encoding::{self, Encoding};
use safeparts_core::{combine_shares, split_secret};

#[test]
fn parse_encoded_shares_from_text() {
    let shares = split_secret(b"synthetic-secret", 2, 3, None).unwrap();
    let encoded = shares
        .iter()
        .map(|share| encoding::encode_packet(share, Encoding::Base64url).unwrap())
        .collect::<Vec<_>>()
        .join("\n");

    let parsed = encoding::parse_share_packets(&encoded, Encoding::Auto).unwrap();
    let recovered = combine_shares(&parsed.packets[..2], None).unwrap();

    assert_eq!(recovered, b"synthetic-secret");
}
```

## اختبار عبارة المرور

غطّ الحالات الثلاث: عبارة المرور الصحيحة، وغياب عبارة المرور، وعبارة المرور الخاطئة. يجب أن تعطي واجهتك إرشاداً آمناً دون إعادة طباعة مدخلات المستخدم.

## أنواع خطأ مفيدة

تعامل مع هذه الحالات صراحة عندما تؤثر على إرشاد المستخدم:

| الخطأ | المعنى |
| --- | --- |
| `InvalidKAndN` | معلمات التقسيم خارج المجال المدعوم. |
| `NotEnoughShares` | تلقى combine أقل من العتبة. |
| `DuplicateX` | تم تقديم إحداثي الحصة نفسه أكثر من مرة. |
| `InconsistentMetadata` | الحصص لا تنتمي إلى set نفسه. |
| `CryptoParamsMismatch` | تختلف metadata التشفير بين الحزم. |
| `IntegrityCheckFailed` | فشلت البايتات المعاد بناؤها في فحص وسم BLAKE3. |
| `PassphraseRequired` | تم دمج حصص مشفرة دون عبارة مرور. |
| `DecryptFailed` | عبارة المرور خاطئة أو البيانات المشفرة عُبث بها. |
| `CouldNotDetectEncoding` | تعذر على الكشف التلقائي استنتاج صيغة الحصة. |

## اختبارات التلف

أتلف نص حصة مرمزة وتأكد من فشل الاسترداد. لا تعتمد على خطأ واحد دقيق لكل أنواع التلف لأن الصيغ المختلفة تفشل في طبقات مختلفة. مثلاً، قد يفشل تلف Base58Check أثناء التحليل، بينما قد تفشل حزمة صحيحة نحوياً مع payload متغير في فحص السلامة.

## الذاكرة والسجلات

لا يستطيع Rust ضمان اختفاء كل نسخة من السر فوراً. أبقِ البايتات المستعادة في أصغر نطاق ممكن، وتجنب تنسيقها كسلاسل نصية، ولا تضع القيم الحساسة في رسائل panic أو سياق الأخطاء.
