يوفّر Safeparts التنفيذ الأساسي في crate باسم `safeparts_core`.
استخدمه عندما يحتاج تطبيق Rust إلى إنشاء حصص أو استرداد سر دون تشغيل CLI كعملية خارجية.

يفترض هذا الدليل أنك مرتاح مع معالجة الأخطاء في Rust، وواجهات API التي تعمل على البايتات، ومخاطر التعامل مع الأسرار داخل ذاكرة العملية.
التركيز هنا على حدّ المكتبة: ما الذي تمرره، وما الذي تستلمه، وأين يجب أن يفرض تطبيقك السياسة التشغيلية.

:::caution
لا تسجّل الأسرار أو النص المسترد أو نصوص الحصص في السجلات.
عامل كل حصة كبيانات حساسة، حتى لو كان العدد المتاح أقل من <var>k</var>.
:::

## متى تستخدم مكتبة Rust

استخدم `safeparts_core` عندما يكون Safeparts جزءاً من منتج أو أداة داخلية:

- سير استرداد داخل خدمة مكتوبة بـ Rust
- أداة إدارية تكتب الحصص إلى مواقع تخزين منفصلة
- أداة نسخ احتياطي تحتاج اختبارات حتمية حول التقسيم والاسترداد
- تطبيق سطح مكتب أو طرفية له واجهته الخاصة

استخدم CLI عندما يكون العمل في الغالب عبر shell أو runbooks أو أتمتة يشغّلها المشغلون.
المكتبة تمنحك تحكماً أكبر، لكنها تجعل تطبيقك مسؤولاً عن الإدخال والتخزين وحدود التدقيق وعمر السر في الذاكرة.

## النموذج الأساسي

واجهة API الرئيسية تعمل على البايتات:

```rust
use safeparts_core::{combine_shares, split_secret, CoreResult};

fn main() -> CoreResult<()> {
    let packets = split_secret(b"database recovery token", 2, 3, None)?;
    let recovered = combine_shares(&packets[..2], None)?;

    assert_eq!(recovered, b"database recovery token");
    Ok(())
}
```

يرجع التقسيم قيماً من نوع `SharePacket`.
تحتوي الحزمة على بيانات وصفية مثل <var>k</var> و<var>n</var> وفهرس الحصة ومعرّف المجموعة، إضافة إلى حمولة الحصة ومعاملات التشفير الاختيارية.
غالباً ستحتاج إلى ترميز الحزم قبل التخزين أو العرض.
