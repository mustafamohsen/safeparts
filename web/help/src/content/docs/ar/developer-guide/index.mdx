استخدم `safeparts_core` عندما يحتاج تطبيق Rust إلى إنشاء حصص Safeparts أو ترميزها أو تحليلها أو فحصها أو استردادها داخل العملية نفسها.

تتعامل المكتبة مع التدفق التشفيري للتقسيم والاسترداد. يبقى على تطبيقك التخزين، والمصادقة، وحدود التدقيق، ومطالبات عبارة المرور، وعمر البايتات الحساسة.

:::caution
لا تسجّل الأسرار أو عبارات المرور أو نص الحصص أو payload الحزم أو البايتات المستعادة. الحصة حساسة حتى عندما يكون لديك أقل من <var>k</var> حصص.
:::

## متى تكون المكتبة هي الحد الصحيح

استخدم `safeparts_core` من أجل:

- سير منتجات يحتاج إلى استرداد بالعتبة؛
- أدوات إدارية داخلية بتخزين مخصص؛
- تطبيقات سطح مكتب أو طرفية بواجهة خاصة؛
- اختبارات تحتاج سلوك تقسيم واسترداد typed؛
- فحص metadata دون طباعة نص الحصص.

استخدم CLI عندما ينتمي سير العمل إلى shell runbook أو إجراء حادثة.

## Round trip بسيط

```rust
use safeparts_core::{combine_shares, split_secret, CoreResult};

fn main() -> CoreResult<()> {
    let packets = split_secret(b"synthetic integration secret", 2, 3, None)?;
    let recovered = combine_shares(&packets[..2], None)?;

    assert_eq!(recovered, b"synthetic integration secret");
    Ok(())
}
```

يعيد التقسيم قيم `SharePacket`. غالباً ترمّز التطبيقات هذه الحزم قبل التخزين أو العرض.
