يمكن أن يكون Safeparts جزءاً من الأتمتة، لكنه لا يجب أن يتحول إلى نظام توزيع أسرار متصل دائماً.
استخدمه للإعداد، والحيازة الاحتياطية، والاسترداد، والإجراءات التي يتحكم بها المشغلون.
استخدم مدير الأسرار للحقن العادي وقت التشغيل.

## نظافة shell

فضّل الملفات والأنابيب ومساحات العمل المقيدة على القيم الحرفية في سطر الأوامر.
لا تستخدم `echo "real-secret"` في الأمثلة أو scripts أو التذاكر.
هذا يشجع تسريب تاريخ shell.

نمط أكثر أماناً:

```bash
umask 077
workdir="$(mktemp -d)"
trap 'rm -rf "$workdir"' EXIT

install -m 600 /dev/null "$workdir/secret.bin"
# اكتب أو انسخ السر إلى $workdir/secret.bin بالطريقة المعتمدة لديك.

safeparts split \
  -k 3 \
  -n 5 \
  -e base64url \
  --in "$workdir/secret.bin" \
  --out "$workdir/shares.txt"
```

بعد ذلك وزع الحصص واحذف المجلد المؤقت.
إذا كانت خطوة التوزيع قد تفشل، فتأكد أنها لا تطبع محتوى الحصص في مسار الخطأ.

## CI وتوقيع الإصدارات

يفيد Safeparts في الحيازة غير المتصلة لمفاتيح توقيع الإصدارات أو مواد الاسترداد.
غالباً لا يكون مناسباً لاسترداد هذه الأسرار داخل وظائف CI عادية.

نمط موصى به:

1. قسّم سر استرداد التوقيع أثناء الإعداد
2. خزّن الحصص مع أمناء منفصلين
3. أبق اعتمادات التوقيع العادية في نظام الإصدارات الآمن الموجود لديك
4. استخدم Safeparts فقط عند الحاجة لاسترداد ذلك النظام أو استبداله
5. دوّر المفتاح بعد الاسترداد إذا جُمعت الحصص على بنية مشتركة

إذا كان لا بد لوظيفة CI أن تشغّل `safeparts combine`، فعامل تلك الوظيفة كبيئة استرداد ذات صلاحيات عالية.
أوقف echo للأوامر، وقيّد السجلات، ونظف ملفات مساحة العمل بعد الاستخدام.

## حد التكامل مع vault

نمط شائع هو تخزين حصة في vault الشركة، وحصة مع قائد الأمن، وحصة غير متصلة.
قد يكون هذا معقولاً لاسترداد <span class="token">k=2, n=3</span>.

تجنب تخزين عدد كاف من الحصص في vault أو نطاق هوية واحد يحقق العتبة.
إذا كان اختراق SSO واحد يمنح الوصول إلى <var>k</var> حصص، فلن يحقق Safeparts فصل الواجبات.

## إخراج مناسب للمعالجة الآلية

يخرج CLI حصة مشفرة واحدة في كل سطر عند التقسيم بترميزات مدمجة.
يمكن للـ scripts تقسيم الإخراج إلى وجهات منفصلة:

```bash
safeparts split -k 3 -n 5 -e base64url --in secret.bin --out shares.txt

sed -n '1p' shares.txt > share-ops.txt
sed -n '2p' shares.txt > share-security.txt
sed -n '3p' shares.txt > share-eng-lead.txt
sed -n '4p' shares.txt > share-offline.txt
sed -n '5p' shares.txt > share-exec.txt
```

اجعل الخطوة التالية تنقل كل ملف إلى موقع الحيازة النهائي.
لا تترك مجلد العمل كحزمة تحتوي كل الحصص.

## قواعد التسجيل

آمن للتسجيل:

- نجاح الأمر أو فشله
- العتبة والترميز المختاران
- أسماء وجهات الحيازة
- معرّفات تذاكر الموافقة

غير آمن للتسجيل:

- بايتات السر المدخل
- الحصص المشفرة
- عبارات المرور
- مخرجات السر المسترد
- محتويات الملفات المؤقتة كاملة

عند الشك، سجّل أقل واطلب ملاحظة استرداد بشرية لا تحتوي مادة سرية.
