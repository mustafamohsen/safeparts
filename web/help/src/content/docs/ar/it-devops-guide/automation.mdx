يمكن أن يكون Safeparts جزءاً من الأتمتة، لكنه لا ينبغي أن يتحول إلى نظام توزيع أسرار online روتيني. استخدمه للإعداد وescrow وتدريبات الاسترداد والاسترداد المعتمد من المشغلين. استخدم مدير أسرار للحقن الروتيني وقت التشغيل.

:::caution
شغّل فحوص CI المتكررة باستخدام أسرار اصطناعية. لا تضع حصصاً إنتاجية أو عبارات مرور أو اعتمادات مستعادة في سجلات CI أو artifacts أو caches أو لقطات شاشة أو تذاكر.
:::

## قواعد scripts

- أبقِ `set -x` معطلاً حول الأوامر التي تلمس الأسرار أو الحصص أو عبارات المرور أو الناتج المستعاد.
- استخدم stdin والملفات بدلاً من القيم الحرفية في سطر الأوامر للمواد الحساسة.
- فضّل `--passphrase-file` على `--passphrase`.
- أنشئ الملفات المؤقتة داخل مجلد خاص مع `umask 077`.
- لا تخزّن عدداً كافياً من الحصص للاسترداد في CI secret store واحد أو bucket artifacts واحد أو repository واحد أو نطاق هوية واحد إلا إذا قبلت السياسة ذلك صراحة.
- لا تطبع الأسرار المستعادة. قارن الملفات أو مرر البايتات مباشرة إلى خطوة واحدة متحكم بها.

## تدريب اصطناعي محلي

يتحقق هذا script من التقسيم والاسترداد دون طباعة القيمة المستعادة.

```bash
#!/usr/bin/env bash
set -euo pipefail
umask 077

workdir="$(mktemp -d)"
trap 'rm -rf "$workdir"' EXIT

printf 'synthetic secret for automation drill' > "$workdir/secret.bin"

safeparts split \
  -k 2 \
  -n 3 \
  -e base64url \
  -i "$workdir/secret.bin" \
  -o "$workdir/shares.txt"

sed -n '1,2p' "$workdir/shares.txt" > "$workdir/selected-shares.txt"

safeparts combine \
  -i "$workdir/selected-shares.txt" \
  -o "$workdir/recovered.bin"

cmp "$workdir/secret.bin" "$workdir/recovered.bin"
```

استخدم هذا النمط لفحوص smoke والتدريبات. لا تستبدل الإدخال الاصطناعي إلا داخل إجراء استرداد مراجع.

## تدريب محمي بعبارة مرور

```bash
#!/usr/bin/env bash
set -euo pipefail
umask 077

workdir="$(mktemp -d)"
trap 'rm -rf "$workdir"' EXIT

printf 'synthetic protected secret' > "$workdir/secret.bin"
printf '%s' 'synthetic passphrase for tests' > "$workdir/passphrase.txt"

safeparts split \
  -k 2 \
  -n 3 \
  -e mnemo-words \
  -P "$workdir/passphrase.txt" \
  -i "$workdir/secret.bin" \
  -o "$workdir/shares.txt"

awk 'NR <= 2 { print }' "$workdir/shares.txt" > "$workdir/selected-shares.txt"

safeparts combine \
  -P "$workdir/passphrase.txt" \
  -i "$workdir/selected-shares.txt" \
  -o "$workdir/recovered.bin"

cmp "$workdir/secret.bin" "$workdir/recovered.bin"
```

## تدريب CI

تدريب CI الآمن يستخدم بيانات اصطناعية ويثبت أن الكود الحالي ما زال ينفذ round-trip.

```yaml
name: safeparts synthetic recovery drill

on:
  workflow_dispatch:

jobs:
  recovery-drill:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable

      - name: Build CLI
        run: cargo build --release -p safeparts

      - name: Run synthetic drill
        shell: bash
        run: |
          set -euo pipefail
          umask 077

          workdir="$(mktemp -d)"
          trap 'rm -rf "$workdir"' EXIT

          printf 'synthetic CI drill secret' > "$workdir/secret.bin"
          target/release/safeparts split -k 2 -n 3 -e base64url -i "$workdir/secret.bin" -o "$workdir/shares.txt"
          sed -n '1,2p' "$workdir/shares.txt" > "$workdir/selected-shares.txt"
          target/release/safeparts combine -i "$workdir/selected-shares.txt" -o "$workdir/recovered.bin"
          cmp "$workdir/secret.bin" "$workdir/recovered.bin"
```

## استرداد حقيقي محروس

استخدم CI للاسترداد الحقيقي فقط بعد مراجعة أمنية. غالباً يوسّع runner الخاص بـ CI نطاق الوصول عبر السجلات ومتغيرات البيئة وcaches وartifacts ورؤية المسؤولين.

إذا كان يجب أن تدمج مهمة CI حصصاً حقيقية:

- شغّلها يدوياً؛
- استخدم runner مقوّى أو self-hosted؛
- تجنّب تخزين <var>k</var> حصص في backend أسرار واحد إلا إذا قبلت السياسة ذلك؛
- أخفِ المتغيرات، لكن لا تعتمد على masking كتحكم أساسي؛
- اكتب الناتج المستعاد إلى ملف خاص؛
- مرر الملف إلى خطوة واحدة مراجعة؛
- احذف الملفات المؤقتة عند الخروج؛
- دوّر الاعتماد المستعاد بعد الاستخدام.
