سر break-glass يُستخدم نادراً، وتحت ضغط، وغالباً بصلاحيات عالية.
يساعد Safeparts على جعل الاسترداد يتطلب عدة أشخاص دون الاعتماد على عنصر vault واحد متصل دائماً.

## مثال: استرداد مسؤول cloud

يريد فريق منصة صغير وصولاً طارئاً إلى حساب cloud إذا تعطلت البنية العادية للهوية.
يختارون <span class="token">k=3, n=5</span>.

حيازة مقترحة:

- الحصة 1: قائد الأمن
- الحصة 2: قائد العمليات
- الحصة 3: مدير الهندسة
- الحصة 4: خزنة شركة غير متصلة
- الحصة 5: راع تنفيذي أو مسؤول امتثال

يتطلب الاسترداد ثلاثة مشاركين مستقلين.
غياب شخص واحد لا يعطل الإجراء، وحامل حصة واحد لا يستطيع الاسترداد وحده.

## إجراء التقسيم

<Steps>
1. أنشئ أو صدّر اعتماد break-glass في بيئة مضبوطة.
2. قسّمه باستخدام CLI والعتبة والترميز المختارين.
3. أعط كل أمين حصة حصته فقط.
4. نفّذ تدريب استرداد باستخدام اعتماد اصطناعي أولاً.
5. نفّذ تحققاً مضبوطاً واحداً بالعملية الحقيقية إذا سمحت السياسة.
6. أتلف الملفات المؤقتة التي تحتوي الاعتماد الأصلي أو كل الحصص.
</Steps>

أمر مثال:

```bash
safeparts split \
  -k 3 \
  -n 5 \
  -e mnemo-words \
  --passphrase-file passphrase.txt \
  --in cloud-admin-recovery.txt \
  --out shares.txt
```

الحصص بالكلمات أطول، لكنها أسهل للقراءة من الورق أثناء حادث.
للتخزين الآلي فقط، يكون `base64url` أبسط غالباً.

## إجراء الاسترداد

<Steps>
1. افتح سجل الحادث أو التغيير الطارئ.
2. أكّد الموافقة حسب السياسة.
3. اجمع <var>k</var> حصص على الأقل في بيئة الاسترداد.
4. ادمج الحصص واسترد الاعتماد.
5. استخدم الاعتماد لمهمة الاسترداد المحددة فقط.
6. دوّر الاعتماد المسترد أو عطّله.
7. أنشئ تقسيماً جديداً في Safeparts للاعتماد البديل.
</Steps>

أمر مثال:

```bash
safeparts combine \
  --passphrase-file passphrase.txt \
  --in recovery-shares.txt \
  --out recovered-cloud-admin.txt
```

:::danger
بعد استرداد break-glass، افترض أن الحصص المجموعة انكشفت.
دوّر الاعتماد الأساسي وأعد التقسيم.
:::

## التدقيق دون تسريب الأسرار

مسار التدقيق المفيد يتضمن:

- من وافق على الاسترداد
- من شارك من الأمناء
- متى بدأ الاسترداد وانتهى
- أي نظام أو اعتماد تم استرداده
- تأكيد أن التدوير حدث بعد ذلك

يجب ألا يتضمن نص الحصص أو عبارات المرور أو لقطات شاشة للحصص أو الاعتمادات المستردة.

## وتيرة التدريبات

نفّذ تدريبات منتظمة باستخدام أسرار اصطناعية.
يجب أن يتأكد التدريب من قدرة الأمناء على العثور على حصصهم، وفهم مسار التواصل، وتشغيل أوامر الاسترداد.
لا تجعل أول تنفيذ كامل للإجراء يحدث أثناء انقطاع خدمة.
