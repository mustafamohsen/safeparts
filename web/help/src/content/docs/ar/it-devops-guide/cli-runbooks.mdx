CLI غير تفاعلي افتراضياً.
يقرأ السر أو الحصص من stdin ما لم تمرر `--in`، ويكتب إلى stdout ما لم تمرر `--out`.
هذا يجعله مناسباً للـ scripts، لكنه يعني أن تاريخ shell وتمريرات الطرفية والسجلات تحتاج انتباهاً.

## التقسيم من ملف

للاعتمادات الحقيقية، فضّل إدخال الملفات على كتابة السر أو استخدام echo في shell.
أنشئ الملف في مكان مضبوط واحذفه بعد التحقق.

```bash
safeparts split \
  --threshold 3 \
  --shares 5 \
  --encoding base64url \
  --in secret.bin \
  --out shares.txt
```

يحتوي ملف الإخراج على حصة واحدة في كل سطر للترميزات المدمجة.
وزّع هذه الأسطر على مواقع حيازة منفصلة.
لا تحتفظ بملف `shares.txt` كاملاً كأثر طويل الأجل.

## الاسترداد من ملف حصص

يمكن أن يقرأ الاسترداد أي <var>k</var> حصص من ملف:

```bash
safeparts combine \
  --in recovery-shares.txt \
  --out recovered-secret.bin
```

إذا فشل اكتشاف الترميز التلقائي، مرره صراحة:

```bash
safeparts combine \
  --encoding base64url \
  --in recovery-shares.txt \
  --out recovered-secret.bin
```

## ملفات عبارات المرور

فضّل `--passphrase-file` على `--passphrase`.
وسائط سطر الأوامر قد تظهر في تاريخ shell، أو فحص العمليات، أو مسجلات الطرفية، أو أدوات التدقيق.

```bash
safeparts split \
  -k 3 \
  -n 5 \
  -e mnemo-words \
  --passphrase-file passphrase.txt \
  --in secret.bin \
  --out shares.txt
```

يجب أن يستخدم الاسترداد عبارة المرور نفسها:

```bash
safeparts combine \
  --passphrase-file passphrase.txt \
  --in recovery-shares.txt \
  --out recovered-secret.bin
```

احم عبارة المرور بشكل منفصل عن الحصص.
إذا كان vault أو الشخص أو الجهاز نفسه يستطيع توفير ما يكفي من الحصص وعبارة المرور، فالعامل الإضافي يصبح إجرائياً في الغالب.

## شكل runbook موصى به

<Steps>
1. أنشئ سراً اصطناعياً واختبر الإجراء كاملاً.
2. قسّم السر الحقيقي من ملف أو مصدر stdin مضبوط.
3. تحقق من الاسترداد باستخدام <var>k</var> حصص بالضبط.
4. وزع الحصص على أشخاص أو vaults أو مواقع مستقلة.
5. احذف الملفات المؤقتة التي احتوت السر الصريح أو كل الحصص معاً.
6. سجّل الحيازة والعتبة والترميز وجهات الاتصال للاسترداد.
</Steps>

## ملفات لا يفضّل إبقاؤها

تجنب الاحتفاظ بهذه الملفات بعد الإعداد إلا إذا سمحت سياستك بذلك صراحة:

- ملف السر الصريح الأصلي
- ملف واحد يحتوي كل الحصص
- تسجيلات طرفية تحتوي نصوص الحصص
- ملف السر المسترد بعد استيراد الاعتماد أو تدويره

إذا احتجت دليلاً تدقيقياً، فسجّل hashes لوثائق الإجراءات غير السرية، والموافقات، والتوقيتات، والمشاركين.
لا تسجل الحصص نفسها.
