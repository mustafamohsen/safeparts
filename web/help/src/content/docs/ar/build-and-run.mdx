استخدم الإصدارات عندما تريد CLI أو TUI أو تطبيق سطح المكتب دون بناء المستودع. ابن من المصدر عندما تطوّر Safeparts أو تستضيف تطبيق الويب الثابت بنفسك.

## تنزيلات الإصدارات

نزّل مخرجات الإصدار من GitHub:

- https://github.com/mustafamohsen/safeparts/releases

تتضمن حزم الإصدار أرشيفات CLI وTUI للمنصات المدعومة، إضافة إلى تطبيقات سطح المكتب:

- Linux: حزم Tauri بصيغ AppImage و`.deb` و`.rpm`
- Windows: مثبتات Tauri بصيغ NSIS `.exe` وMSI
- معاينة Windows 11: أرشيفات `.zip` ذاتية الاكتفاء لتطبيق WinUI الأصلي، بإصداري x64 وARM64، من دون توقيع
- macOS 14 أو أحدث: ملف `.dmg` عام لتطبيق SwiftUI الأصلي، من دون توقيع

فحص سريع بعد تثبيت binaries الخاصة بـ CLI/TUI:

<Steps>
1. ضع binaries على <span class="token token--mono" dir="ltr">PATH</span>.
2. شغّل <span class="token token--mono" dir="ltr">safeparts --help</span>.
3. شغّل <span class="token token--mono" dir="ltr">safeparts-tui --help</span>.
</Steps>

### Gatekeeper في macOS

إذا عزل macOS ملفاً تم تنزيله وكنت تثق بمخرج الإصدار، أزل خاصية العزل:

```bash
xattr -dr com.apple.quarantine /path/to/safeparts-release/
```

أو أزلها من التطبيق الأصلي بعد نسخه إلى Applications:

```bash
xattr -dr com.apple.quarantine /Applications/Safeparts.app
```

ملف DMG الأصلي غير موقّع وغير موثّق لدى Apple. لا تتجاوز Gatekeeper إلا إذا نزّلت الملف من إصدار المشروع وتحققت من checksum.

### معاينة Windows الأصلية وSmartScreen

اختر أرشيف المعاينة x64 أو ARM64 المطابق لجهاز Windows 11. الأرشيفات الأصلية غير موقعة، لذلك قد يعرض SmartScreen تحذيراً عند التشغيل الأول. تحقق من ملف zip باستخدام `SHA256SUMS.txt`، ثم فك الأرشيف كاملاً وأبق ملفات runtime بجوار `Safeparts.exe`. لا تستبدل المعاينة تطبيق Tauri المثبت.

## بناء binaries من المصدر

المتطلبات:

- Rust stable
- `rustfmt` و`clippy`

```bash
cargo build
cargo test --all-features
```

فحص lint مشابه لـ CI:

```bash
cargo fmt --all -- --check
cargo clippy --all-targets --all-features -- -D warnings
```

Binaries الإصدار:

```bash
cargo build --release -p safeparts -p safeparts_tui
```

## تطبيق الويب

يوجد تطبيق الويب في <span class="token token--mono" dir="ltr">web/</span>. يحتاج إلى خطوة بناء WASM.

المتطلبات:

- Bun
- `wasm-pack`، أو دع script البناء يثبته

```bash
cd web
bun install
bun run build:wasm
bun run dev
```

افتح [http://localhost:5173](http://localhost:5173).

## تطبيق Tauri لسطح المكتب

يوجد تطبيق Tauri المستخدم في إصدارات Linux وWindows في <span class="token token--mono" dir="ltr">desktop/</span>.

```bash
cd desktop
bun install
bun run tauri:dev
```

لبناء الواجهة:

```bash
bun run build
```

لبناء حزمة Tauri لمنصتك:

```bash
bun run tauri:build
```

## تطبيق macOS الأصلي

على macOS مع Xcode وSwift 6 وRust:

```bash
mise run macos:check
swift run --package-path macos SafepartsMac
```

لبناء ملف DMG العام وغير الموقّع:

```bash
RELEASE_VERSION=v0.3.0 mise run macos:package
```

## معاينة تطبيق Windows الأصلي

على Windows 11 مع .NET 10 وRust:

```powershell
python windows/scripts/prepare.py --check
cargo build -p safeparts_uniffi
dotnet run --project windows/Safeparts.App/Safeparts.App.csproj -p:Platform=x64 -p:RuntimeIdentifier=win-x64
```

تُبنى حزمة المعاينة على المعمارية المطابقة:

```powershell
python windows/scripts/package-release.py 0.3.0 x64
# شغّل هذا الأمر على Windows ARM64 لبناء حزمة ARM64:
python windows/scripts/package-release.py 0.3.0 arm64
```

## تطبيق الويب والوثائق عبر Docker

يبني Dockerfile في المستودع WASM وتطبيق الويب وموقع المساعدة، ثم يخدم المخرجات الثابتة عبر Nginx.

```bash
docker build -t safeparts-webui -f web/Dockerfile .
docker run --rm -p 8080:8080 safeparts-webui
```

افتح [http://localhost:8080](http://localhost:8080). الوثائق على [http://localhost:8080/help/](http://localhost:8080/help/).

## موقع المساعدة

يوجد موقع المساعدة في <span class="token token--mono" dir="ltr">web/help/</span> ويُخدم تحت <span class="token token--mono" dir="ltr">/help/</span>.

```bash
cd web/help
bun install
bun run dev
```

افتح [http://localhost:4321/help/](http://localhost:4321/help/).

لبناء موقع المساعدة الثابت:

```bash
bun run build
```
