ابدأ بتجربة تدريبية. تجربة قصيرة تعلّمك سير العمل دون تعريض سر حقيقي للخطر.

:::caution
تدرّب باستخدام سر اصطناعي مثل <span class="token" dir="ltr">synthetic-secret</span>. لا تستخدم مفتاحاً إنتاجياً قبل اختبار الاسترداد.
:::

## نموذج الاسترداد

تختار رقمين:

- <var>n</var>: عدد الحصص التي ينشئها Safeparts.
- <var>k</var>: عدد الحصص المطلوب لاسترداد السر.

مع أقل من <var>k</var> حصص، يجب ألا يكشف الاسترداد معلومات مفيدة. ومع أي <var>k</var> حصص من نفس عملية التقسيم، ينجح الاسترداد.

تدعم النواة المجال <span class="token" dir="ltr">1 &lt;= k &lt;= n &lt;= 255</span>. في خطط الاسترداد الحقيقية، استخدم <span class="token" dir="ltr">k >= 2</span> حتى لا يستطيع حامل واحد الاسترداد وحده.

## نفّذ التجربة

تطبيق الويب هو أسرع طريقة لتعلّم السير.

<LinkButton href="/" icon="external" target="_blank" rel="noopener noreferrer">
	افتح تطبيق الويب
</LinkButton>

<Steps>
1. افتح تبويب <span class="token">تقسيم</span>.
2. أدخل <span class="token" dir="ltr">synthetic-secret</span> كسر.
3. عيّن <span class="token" dir="ltr">k=2</span> و<span class="token" dir="ltr">n=3</span>.
4. اختر <span class="token">Words</span> إذا كنت تريد قراءة الحصص أو طباعتها، أو <span class="token">Letters</span> إذا كنت تريد نصاً مضغوطاً.
5. اضغط <span class="token">تقسيم</span> وانسخ الحصص الثلاث إلى ملاحظات مؤقتة.
6. افتح تبويب <span class="token">استعادة</span> واستعد السر باستخدام أي حصتين.
7. احذف الحصص والملاحظات الاصطناعية.
</Steps>

## قبل استخدام سر حقيقي

1. اختر خطة استرداد يمكن تنفيذها تحت الضغط.
2. خزّن الحصص في نطاقات فشل منفصلة: أشخاص أو حسابات أو أجهزة أو مواقع مختلفة.
3. اكتب من يحمل كل حصة وكيف تتم الموافقة على الاسترداد.
4. نفّذ تدريباً واحداً ببيانات اصطناعية.
5. اقرأ [نموذج الأمان](../security/) قبل التعامل مع مادة إنتاجية.
