Use releases when you want the CLI, TUI, or desktop app without building the repository. Build from source when you are developing Safeparts or self-hosting the static web app.

## Release downloads

Download release artifacts from GitHub:

- https://github.com/mustafamohsen/safeparts/releases

Release packaging includes CLI and TUI archives for supported platforms. It also includes desktop apps:

- Linux: Tauri AppImage, `.deb`, and `.rpm`
- Windows: Tauri NSIS `.exe` and MSI installers
- Windows 11 preview: unsigned self-contained native WinUI `.zip` archives for x64 and ARM64
- macOS 14+: unsigned universal native SwiftUI `.dmg`

Quick check after installing CLI/TUI binaries:

<Steps>
1. Put the binaries on your <span class="token token--mono">PATH</span>.
2. Run <span class="token token--mono">safeparts --help</span>.
3. Run <span class="token token--mono">safeparts-tui --help</span>.
</Steps>

### macOS Gatekeeper

If macOS quarantines a downloaded binary and you trust the release artifact, remove the quarantine attribute:

```bash
xattr -dr com.apple.quarantine /path/to/safeparts-release/
```

Or remove it from the native app after copying it to Applications:

```bash
xattr -dr com.apple.quarantine /Applications/Safeparts.app
```

The native DMG is unsigned and unnotarized. Only override Gatekeeper when you downloaded the artifact from the project release and verified its checksum.

### Native Windows preview and SmartScreen

Choose the x64 or ARM64 preview archive that matches your Windows 11 device. The native archives are unsigned, so SmartScreen may warn on first launch. Verify the downloaded zip against `SHA256SUMS.txt`, extract the complete directory, and keep its runtime files beside `Safeparts.exe`. The preview does not replace an installed Tauri app.

## Build Rust binaries from source

Prerequisites:

- Rust stable
- `rustfmt` and `clippy`

```bash
cargo build
cargo test --all-features
```

CI-style lint:

```bash
cargo fmt --all -- --check
cargo clippy --all-targets --all-features -- -D warnings
```

Release binaries:

```bash
cargo build --release -p safeparts -p safeparts_tui
```

## Web app

The web app lives in <span class="token token--mono">web/</span>. It requires a WASM build step.

Prerequisites:

- Bun
- `wasm-pack`, or allow the build script to install it

```bash
cd web
bun install
bun run build:wasm
bun run dev
```

Open [http://localhost:5173](http://localhost:5173).

## Tauri desktop app

The Tauri app used for Linux and Windows releases lives in <span class="token token--mono">desktop/</span>.

```bash
cd desktop
bun install
bun run tauri:dev
```

Build the frontend:

```bash
bun run build
```

Build a Tauri bundle for your host platform:

```bash
bun run tauri:build
```

## Native macOS app

On macOS with Xcode, Swift 6, and Rust:

```bash
mise run macos:check
swift run --package-path macos SafepartsMac
```

Build the unsigned universal release DMG:

```bash
RELEASE_VERSION=v0.3.0 mise run macos:package
```

## Native Windows preview

On Windows 11 with .NET 10 and Rust:

```powershell
python windows/scripts/prepare.py --check
cargo build -p safeparts_uniffi
dotnet run --project windows/Safeparts.App/Safeparts.App.csproj -p:Platform=x64 -p:RuntimeIdentifier=win-x64
```

Preview packaging runs on the matching native architecture:

```powershell
python windows/scripts/package-release.py 0.3.0 x64
# Run this on Windows ARM64 for the ARM64 package:
python windows/scripts/package-release.py 0.3.0 arm64
```

## Web app and docs with Docker

The repository Dockerfile builds WASM, the web app, and the help site, then serves the static output through Nginx.

```bash
docker build -t safeparts-webui -f web/Dockerfile .
docker run --rm -p 8080:8080 safeparts-webui
```

Open [http://localhost:8080](http://localhost:8080). Docs are at [http://localhost:8080/help/](http://localhost:8080/help/).

## Help site

The help site lives in <span class="token token--mono">web/help/</span> and is served under <span class="token token--mono">/help/</span>.

```bash
cd web/help
bun install
bun run dev
```

Open [http://localhost:4321/help/](http://localhost:4321/help/).

Build the static help site:

```bash
bun run build
```
