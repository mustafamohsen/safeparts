Start with a practice run. A short drill teaches the workflow without exposing a real secret.

Use a synthetic secret such as <span class="token">synthetic-secret</span>. Save production keys for after you have tested recovery.

## The recovery model

Plans are usually described as "2 of 3" or "3 of 5".

A 2-of-3 plan creates three shares. Any two can recover the secret. One lost share is fine, and one holder alone cannot recover.

In commands and technical pages, the required count is called the threshold and is written as <var>k</var>. The total share count is written as <var>n</var>. The core supports <span class="token" dir="ltr">1 &lt;= k &lt;= n &lt;= 255</span>, but real recovery plans usually use a threshold of at least two.

## Run the drill

The web app is the fastest way to learn the flow.

<LinkButton href="/" icon="external" target="_blank" rel="noopener noreferrer">
	Open the web app
</LinkButton>

<Steps>
1. Open the <span class="token">Split</span> tab.
2. Enter <span class="token">synthetic-secret</span> as the secret.
3. Set the required shares to <span class="token">2</span> and total shares to <span class="token">3</span>.
4. Choose <span class="token">Words</span> if you want to read or print the shares, or <span class="token">Letters</span> if you want compact text.
5. Click <span class="token">Split</span> and copy the three shares into temporary notes.
6. Open the <span class="token">Combine</span> tab and recover the secret with any two shares.
7. Delete the synthetic shares and notes.
</Steps>

## Before using a real secret

1. Choose a recovery plan you can run under stress.
2. Store shares in separate failure domains: different people, accounts, devices, or locations.
3. Write down who holds each share and how recovery is approved.
4. Run one recovery drill with synthetic data.
5. Read the [security model](../security/) before handling production material.
