---
title: Encodings and packets
description: Store, parse, and inspect SharePacket values safely.
---

A split returns binary `SharePacket` values. Most applications should encode packets before storage, display, or transfer.

## Encode packets

Use `safeparts_core::encoding` for the public text encoding API.

```rust
use safeparts_core::encoding::{self, Encoding};
use safeparts_core::{split_secret, CoreResult};

fn encode_for_storage() -> CoreResult<Vec<String>> {
    let packets = split_secret(b"synthetic storage secret", 2, 3, None)?;

    packets
        .iter()
        .map(|packet| encoding::encode_packet(packet, Encoding::Base64url))
        .collect()
}
```

`Encoding::Auto` is for parsing only. Output must choose one concrete encoding.

## Parse pasted input

```rust
use safeparts_core::encoding::{self, Encoding};
use safeparts_core::{combine_shares, CoreResult};

fn recover_from_paste(input: &str) -> CoreResult<Vec<u8>> {
    let parsed = encoding::parse_share_packets(input, Encoding::Auto)?;
    combine_shares(&parsed.packets, None)
}
```

For UI text areas where long mnemonic shares may wrap across lines, use `parse_share_packets_wrapped_mnemonics`. Blank lines still separate mnemonic shares.

## Supported encodings

| Encoding | Use |
| --- | --- |
| `Encoding::Base64url` | Compact URL-safe text. |
| `Encoding::Base58check` | Compact text with Base58Check checksum. |
| `Encoding::MnemoWords` | Word groups with CRC16 for manual transcription. |
| `Encoding::MnemoBip39` | BIP-39-valid word sequences for compatible storage workflows. |
| `Encoding::Auto` | Parse-only detection for operator input. |

## Packet metadata

Each `SharePacket` contains:

- `set_id`: random identifier for the split set;
- `k` and `n`: threshold and total share count;
- `x`: share coordinate;
- `payload`: share bytes;
- `crypto_params`: optional Argon2id and nonce parameters for encrypted splits.

The metadata is not secret. It lets combine reject mixed sets, duplicate indexes, and incompatible crypto parameters.

## Storage pattern

A typical application flow:

1. Split bytes into `SharePacket` values.
2. Encode each packet with one concrete encoding.
3. Store each encoded share in a separate failure domain.
4. Store the encoding name with custody metadata if operators will not use auto-detection.
5. Later parse at least <var>k</var> encoded shares.
6. Combine packets and handle recovered bytes as sensitive output.

Do not keep enough shares together in one database row, vault item, CI secret store, or log stream unless policy treats that location like the original secret.
