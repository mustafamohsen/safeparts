Use `safeparts_core` when a Rust application needs to create, encode, parse, inspect, or recover Safeparts shares inside its own process.

The library handles the cryptographic split and combine flow. Your application still owns storage, authentication, audit boundaries, passphrase prompts, and the lifetime of sensitive bytes.

:::caution
Do not log secrets, passphrases, share text, packet payloads, or recovered bytes. A share is sensitive even when fewer than <var>k</var> shares are present.
:::

## When the library is the right boundary

Use `safeparts_core` for:

- product workflows that need threshold recovery;
- internal admin tools with custom storage;
- desktop or terminal apps with their own UI;
- tests that need typed split and combine behavior;
- metadata inspection without printing share text.

Use the CLI when the workflow belongs in a shell runbook or incident procedure.

## Minimal round trip

```rust
use safeparts_core::{combine_shares, split_secret, CoreResult};

fn main() -> CoreResult<()> {
    let packets = split_secret(b"synthetic integration secret", 2, 3, None)?;
    let recovered = combine_shares(&packets[..2], None)?;

    assert_eq!(recovered, b"synthetic integration secret");
    Ok(())
}
```

A split returns `SharePacket` values. Most applications encode those packets before storage or display.
