---
title: Testing and errors
description: Test Safeparts integrations with synthetic data and typed failure cases.
---

Tests should prove your integration boundaries without using production secrets. Synthetic byte strings are enough to cover split, combine, encoding, corruption, and passphrase behavior.

## Test threshold behavior

```rust
use safeparts_core::{combine_shares, split_secret, CoreError};

#[test]
fn round_trip_with_any_k_shares() {
    let shares = split_secret(b"synthetic-secret", 2, 3, None).unwrap();

    let recovered = combine_shares(&[shares[0].clone(), shares[2].clone()], None).unwrap();

    assert_eq!(recovered, b"synthetic-secret");
}

#[test]
fn insufficient_shares_fail() {
    let shares = split_secret(b"synthetic-secret", 3, 5, None).unwrap();

    let err = combine_shares(&shares[..2], None).unwrap_err();

    assert!(matches!(err, CoreError::NotEnoughShares { .. }));
}
```

## Test encoding boundaries

If your application stores text shares, test encoding and parsing together.

```rust
use safeparts_core::encoding::{self, Encoding};
use safeparts_core::{combine_shares, split_secret};

#[test]
fn parse_encoded_shares_from_text() {
    let shares = split_secret(b"synthetic-secret", 2, 3, None).unwrap();
    let encoded = shares
        .iter()
        .map(|share| encoding::encode_packet(share, Encoding::Base64url).unwrap())
        .collect::<Vec<_>>()
        .join("\n");

    let parsed = encoding::parse_share_packets(&encoded, Encoding::Auto).unwrap();
    let recovered = combine_shares(&parsed.packets[..2], None).unwrap();

    assert_eq!(recovered, b"synthetic-secret");
}
```

## Test passphrase behavior

Cover all three cases: correct passphrase, missing passphrase, and wrong passphrase. Your UI should give safe guidance without echoing user input.

## Useful error classes

Handle these explicitly when they affect user guidance:

| Error | Meaning |
| --- | --- |
| `InvalidKAndN` | Split parameters are outside the supported range. |
| `NotEnoughShares` | Combine received fewer than the threshold. |
| `DuplicateX` | The same share coordinate was supplied more than once. |
| `InconsistentMetadata` | Shares do not belong to the same set. |
| `CryptoParamsMismatch` | Encrypted share metadata differs across packets. |
| `IntegrityCheckFailed` | Reconstructed bytes failed the BLAKE3 tag check. |
| `PassphraseRequired` | Encrypted shares were combined without a passphrase. |
| `DecryptFailed` | Passphrase is wrong or encrypted data was tampered with. |
| `CouldNotDetectEncoding` | Auto-detection could not infer the share format. |

## Corruption tests

Corrupt encoded share text and assert that recovery fails. Do not require one exact error for all corruptions because different formats fail at different layers. For example, Base58Check corruption may fail during decoding, while a syntactically valid packet with changed payload bytes may fail the integrity check.

## Memory and logging

Rust cannot guarantee that every copy of a secret disappears immediately. Keep recovered bytes in the smallest scope you can, avoid formatting them into strings, and do not include sensitive values in panic messages or error context.
