---
title: How Safeparts works
description: A concise explanation of threshold shares, packets, integrity checks, and optional encryption.
---

Safeparts has one job: split a secret so recovery requires enough matching shares.

## Threshold sharing

You choose <var>k</var> and <var>n</var>. Safeparts creates <var>n</var> shares, and any <var>k</var> of them can reconstruct the original secret.

Under the hood, Safeparts uses Shamir-style secret sharing over <span class="token token--mono">GF(256)</span>. The scheme works on bytes, so the input can be text, a binary key, or another byte sequence.

A share is not a slice of the secret. Conceptually, each share is a point needed to reconstruct a polynomial. Fewer than <var>k</var> points do not reveal useful information about the secret.

## Share packets

Safeparts wraps each raw share in a versioned packet. The packet includes:

- threshold and share count: <var>k</var> and <var>n</var>
- share index: <var>x</var>
- random set ID for the split
- share payload bytes
- optional encryption parameters when a passphrase was used

This metadata is not secret. It helps the combine path reject duplicate shares, mixed share sets, and incompatible passphrase parameters.

## Integrity checks

Secret sharing alone does not reliably tell you when an operator typed a share wrong or mixed shares from different splits. Safeparts appends a BLAKE3 integrity tag before splitting.

On recovery, Safeparts reconstructs the tagged bytes and verifies the tag. If shares are corrupted or mismatched, recovery fails instead of returning silent garbage.

## Optional passphrase protection

When you enter a passphrase, Safeparts encrypts the secret before splitting it:

1. Derive a key from the passphrase with Argon2id.
2. Encrypt the secret with ChaCha20-Poly1305.
3. Add the BLAKE3 integrity tag.
4. Split the tagged ciphertext into shares.

Recovery then needs both enough shares and the passphrase.

## Limits

The core threshold range is <span class="token" dir="ltr">1 &lt;= k &lt;= n &lt;= 255</span>. Real recovery plans should normally use <span class="token">k >= 2</span> so one holder cannot recover alone.

Safeparts does not store, sync, refresh, or rotate shares for you. If you need to change holders or recover after exposure, reconstruct the secret, rotate the underlying credential if possible, and create a new split.
