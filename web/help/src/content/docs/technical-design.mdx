## Data flow

Safeparts operates on bytes.

```text
Split
-----
secret bytes
  |
  | optional passphrase protection
  |   Argon2id(passphrase, salt, params) -> key
  |   ChaCha20-Poly1305(key, nonce) -> ciphertext
  |
  | data_to_split = plaintext or ciphertext
  |
  | BLAKE3(data_to_split) -> 32-byte tag
  | tagged = data_to_split || tag
  |
  | Shamir split over GF(256), byte by byte
  |
  | SharePacket per share
  |
  +-> text encoding for storage or transport
```

```text
Combine
-------
text input -> SharePacket values
  |
  | validate set metadata, share indexes, and crypto params
  |
  | Shamir combine over GF(256)
  |
  | split reconstructed bytes into data_to_split and tag
  | verify BLAKE3(data_to_split) == tag
  |
  | if encrypted:
  |   Argon2id(passphrase, salt, params) -> key
  |   ChaCha20-Poly1305 decrypt -> secret bytes
  |
  +-> recovered secret bytes
```

## Threshold sharing

Safeparts uses Shamir-style secret sharing over <span class="token token--mono">GF(256)</span>, applied independently to each byte position.

For each byte in the tagged payload:

1. Treat the byte as the constant term of a polynomial.
2. Sample <var>k</var>-1 random coefficients.
3. Evaluate the polynomial at share coordinates <span class="token" dir="ltr">x = 1..n</span>.
4. Store each result byte in that share's payload.

A share payload is the same length as the tagged payload. Reconstruction uses any <var>k</var> unique share coordinates from the same set and interpolates each byte back at <span class="token" dir="ltr">x = 0</span>.

The core range is <span class="token" dir="ltr">1 &lt;= k &lt;= n &lt;= 255</span>. The field has 256 values and Safeparts reserves <span class="token" dir="ltr">x = 0</span> for reconstruction.

## Share packets

A `SharePacket` is the versioned envelope around one raw share. It carries non-secret metadata needed to reject common mistakes.

Current binary packet shape, simplified:

```text
magic       : "SMN1"
version     : u8
flags       : u8
k, n, x     : u8, u8, u8
set_id      : 16 bytes

if encrypted:
  salt          : 16 bytes
  nonce         : 12 bytes
  argon memory  : u32 KiB
  argon time    : u32
  argon lanes   : u32

payload_len : u32
payload     : bytes
```

Version 2 packets can carry encryption parameters. Version 1 packets remain decodable for compatibility where supported by the packet parser.

Packet metadata is not secret. Security comes from threshold sharing and, when enabled, encryption before splitting.

## Integrity tag

Shamir sharing can reconstruct output from the wrong inputs. Safeparts appends a 32-byte BLAKE3 tag to the data before splitting so combine can reject corrupted or mixed shares.

The tag detects:

- damaged text or packet bytes;
- shares from different split sets;
- transcription errors that pass the text encoding layer;
- reconstructed ciphertext that does not match the original encrypted payload.

The BLAKE3 tag is not a keyed MAC. It is an internal consistency check, not proof that a trusted party created the shares.

## Passphrase protection

Passphrase protection encrypts once before splitting. The passphrase never appears in a packet.

Current key derivation and encryption:

- KDF: Argon2id.
- Salt: 16 random bytes per split.
- Default memory cost: 65,536 KiB.
- Default time cost: 3.
- Default parallelism: 1.
- AEAD: ChaCha20-Poly1305.
- Nonce: 12 random bytes per split.

If an attacker obtains <var>k</var> encrypted shares, they can reconstruct the ciphertext and attempt offline passphrase guesses. Argon2id raises the cost of that attack, but weak passphrases remain weak.

Safeparts verifies the BLAKE3 tag before decrypting. For encrypted splits, that verifies the reconstructed ciphertext. ChaCha20-Poly1305 then rejects wrong passphrases or tampered ciphertext.

## Text encodings

The binary packet can be encoded as:

- <span class="token">base64url</span>
- <span class="token">base58check</span>
- <span class="token">mnemo-words</span>
- <span class="token">mnemo-bip39</span>

The encoding layer is operational. It helps storage, transport, and manual entry. It does not change the underlying threshold scheme.

See [encodings](../encodings/) for surface support and selection guidance.

## Failure behavior

Combine fails when:

- fewer than <var>k</var> shares are supplied;
- duplicate share coordinates are supplied;
- packet metadata is inconsistent;
- crypto parameters differ across encrypted shares;
- the reconstructed integrity tag does not match;
- encrypted shares require a passphrase and none is provided;
- decryption fails.

Applications should surface the error class without logging share text, passphrases, or recovered bytes.

## Limitations

- No storage layer. Safeparts does not decide where shares live.
- No verifiable secret sharing. A malicious holder can still provide a bad share and cause recovery to fail.
- No share refresh without reconstructing and creating a new split.
- No protection against a legitimate quorum. Anyone with <var>k</var> valid shares and the passphrase, if used, can recover.
- No wallet seed semantics. Mnemonic shares are Safeparts share encodings, not wallet seeds.
