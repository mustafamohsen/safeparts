---
title: Automation patterns
description: Use Safeparts in scripts and CI drills without leaking secrets into logs.
---

Safeparts can be part of automation, but it should not become your routine online secret distribution system. Use it for setup, escrow, recovery drills, and operator-approved recovery. Use a secrets manager for normal runtime injection.

:::caution
Run recurring CI checks with synthetic secrets. Do not put production shares, passphrases, or recovered credentials in CI logs, artifacts, caches, screenshots, or tickets.
:::

## Rules for scripts

- Keep `set -x` off around commands that touch secrets, shares, passphrases, or recovered output.
- Use stdin and files rather than command-line literals for secret material.
- Prefer `--passphrase-file` over `--passphrase`.
- Create temporary files under a private directory with `umask 077`.
- Do not store enough shares to recover in one CI secret store, artifact bucket, repository, or identity domain unless that risk is explicitly accepted.
- Do not print recovered secrets. Compare files or pass bytes directly to one controlled step.

## Local synthetic drill

This script verifies split and combine without printing the recovered value.

```bash
#!/usr/bin/env bash
set -euo pipefail
umask 077

workdir="$(mktemp -d)"
trap 'rm -rf "$workdir"' EXIT

printf 'synthetic secret for automation drill' > "$workdir/secret.bin"

safeparts split \
  -k 2 \
  -n 3 \
  -e base64url \
  -i "$workdir/secret.bin" \
  -o "$workdir/shares.txt"

sed -n '1,2p' "$workdir/shares.txt" > "$workdir/selected-shares.txt"

safeparts combine \
  -i "$workdir/selected-shares.txt" \
  -o "$workdir/recovered.bin"

cmp "$workdir/secret.bin" "$workdir/recovered.bin"
```

Use this pattern for smoke tests and drills. Replace the synthetic input only inside a reviewed recovery procedure.

## Passphrase-protected drill

```bash
#!/usr/bin/env bash
set -euo pipefail
umask 077

workdir="$(mktemp -d)"
trap 'rm -rf "$workdir"' EXIT

printf 'synthetic protected secret' > "$workdir/secret.bin"
printf '%s' 'synthetic passphrase for tests' > "$workdir/passphrase.txt"

safeparts split \
  -k 2 \
  -n 3 \
  -e mnemo-words \
  -P "$workdir/passphrase.txt" \
  -i "$workdir/secret.bin" \
  -o "$workdir/shares.txt"

awk 'NR <= 2 { print }' "$workdir/shares.txt" > "$workdir/selected-shares.txt"

safeparts combine \
  -P "$workdir/passphrase.txt" \
  -i "$workdir/selected-shares.txt" \
  -o "$workdir/recovered.bin"

cmp "$workdir/secret.bin" "$workdir/recovered.bin"
```

## CI drill

A safe CI drill uses synthetic data and proves that the checked-in code can still round-trip.

```yaml
name: safeparts synthetic recovery drill

on:
  workflow_dispatch:

jobs:
  recovery-drill:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable

      - name: Build CLI
        run: cargo build --release -p safeparts

      - name: Run synthetic drill
        shell: bash
        run: |
          set -euo pipefail
          umask 077

          workdir="$(mktemp -d)"
          trap 'rm -rf "$workdir"' EXIT

          printf 'synthetic CI drill secret' > "$workdir/secret.bin"
          target/release/safeparts split -k 2 -n 3 -e base64url -i "$workdir/secret.bin" -o "$workdir/shares.txt"
          sed -n '1,2p' "$workdir/shares.txt" > "$workdir/selected-shares.txt"
          target/release/safeparts combine -i "$workdir/selected-shares.txt" -o "$workdir/recovered.bin"
          cmp "$workdir/secret.bin" "$workdir/recovered.bin"
```

## Guarded real recovery

Use CI for real recovery only after a security review. A CI runner often expands access through logs, environment variables, caches, artifacts, and administrator visibility.

If a job must combine real shares:

- trigger it manually;
- use a hardened or self-hosted runner;
- avoid storing <var>k</var> shares in one secret backend unless accepted by policy;
- mask variables, but do not rely on masking as the primary control;
- write recovered output to a private file;
- pass the file to one reviewed step;
- delete temporary files on exit;
- rotate the recovered credential after use.

## When to use the Rust library

Use `safeparts_core` instead of shell automation when you need typed errors, metadata inspection, custom storage, in-process prompts, or tighter logging controls.

See [library API](../../developer-guide/library-api/).
