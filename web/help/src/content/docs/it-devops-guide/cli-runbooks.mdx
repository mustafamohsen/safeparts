The CLI is non-interactive unless your runbook adds prompts around it. That makes it reliable in procedures, but it also means shell history, terminal scrollback, and temporary files matter.

## Split from a controlled file

Use file input for real credentials when possible. Create the file in a private working directory and remove it after verification.

```bash
safeparts split \
  --threshold 3 \
  --shares 5 \
  --encoding mnemo-words \
  --in secret.bin \
  --out shares.txt
```

Distribute each share from `shares.txt` to its assigned holder or location. Do not keep `shares.txt` as a long-term artifact unless it is protected like the original secret.

## Combine selected shares

Recovery should use only the shares gathered for the approved event.

```bash
safeparts combine \
  --in selected-shares.txt \
  --out recovered-secret.bin
```

If needed, pin the encoding:

```bash
safeparts combine \
  --encoding mnemo-words \
  --in selected-shares.txt \
  --out recovered-secret.bin
```

## Passphrase file pattern

Prefer a passphrase file over a command-line literal.

```bash
safeparts split \
  -k 3 \
  -n 5 \
  -e base64url \
  --passphrase-file passphrase.txt \
  --in secret.bin \
  --out shares.txt
```

Recovery uses the same passphrase source:

```bash
safeparts combine \
  --passphrase-file passphrase.txt \
  --in selected-shares.txt \
  --out recovered-secret.bin
```

Store the passphrase through a separate policy path. Do not put it next to enough shares to recover.

## Runbook template

<Steps>
1. State the secret name, owner, threshold, encoding, and passphrase policy.
2. Define who approves split and recovery.
3. Define the working directory and cleanup commands.
4. Split a synthetic secret with the same commands.
5. Split the real secret.
6. Verify recovery if policy allows it.
7. Distribute shares and record custody without recording share text.
8. Delete temporary files that contain plaintext, all shares, passphrases, or recovered output.
9. Schedule a drill and a custody review.
</Steps>

## Files to avoid keeping

Avoid retaining:

- the original plaintext file;
- a file containing all shares;
- passphrase files outside the approved passphrase path;
- recovered secret files after the secret has been imported or rotated;
- terminal transcripts or screenshots containing sensitive material.

For audit, record approvals, participants, timestamps, command versions, and rotation status.
